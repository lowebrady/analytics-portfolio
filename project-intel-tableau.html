08 — Data Visualization / Tableau

Intel Energy
Dashboard

Built a multi-sheet Tableau dashboard for Intel's Sustainability Team to support data center site selection — combining two datasets into an interactive decision-support tool with calculated fields, dynamic parameters, and drill-down filtering across multiple chart types.

Role Data Visualization Analyst
Tools Tableau Desktop, Calculated Fields, Parameters
Domain Energy / Sustainability / Operations
Context Global Career Accelerator — MSU
01 — The Business Problem

Data center site selection is a multi-million dollar decision

Data centers are among the most energy-intensive facilities on earth. For a company like Intel, choosing where to site a new data center involves balancing dozens of variables — energy costs, grid reliability, renewable energy availability, water access for cooling, and local incentive structures. Making that call based on gut instinct or static spreadsheets leaves significant value on the table.

Intel's Sustainability Team needed a tool that could bring together multiple data sources and let analysts explore tradeoffs interactively — rather than generating a new report every time a stakeholder asked a different question. The goal was to build a dashboard that made the data explorable and the decision-making process faster and more transparent.

"A great dashboard doesn't just display data — it changes the questions people think to ask. The goal is to make exploration feel effortless so analysts spend their time thinking, not formatting."

02 — Dashboard Design & Build

Multi-sheet architecture for a multi-dimensional decision

The dashboard was designed around the actual decision workflow: start with a high-level geographic view, drill into regional comparisons, then examine specific site-level metrics. Each sheet in the workbook serves a distinct analytical purpose rather than duplicating information.

  • 01
    Data Integration Combined two separate datasets — regional energy pricing and infrastructure availability data — using Tableau's data blending functionality. Aligned on a shared geographic key to enable cross-dataset calculations without losing granularity in either source.
  • 02
    Calculated Fields Built custom calculated fields to derive metrics not present in the raw data: cost-per-megawatt estimates, renewable energy percentage by region, and a composite site-score index that weighted multiple factors according to Intel's stated priorities. These fields update dynamically as filters change.
  • 03
    Chart Type Selection Matched chart types to the analytical questions each sheet was answering. Bar charts for direct regional comparisons. Dual-axis plots to show the relationship between energy cost and renewable availability on the same scale. Tree maps for visualizing relative energy consumption by facility type. Line charts for tracking grid reliability trends over time.
  • 04
    Dynamic Parameters & Filters Added dropdown parameters allowing users to weight the composite site score by their own priorities — for example, a user who prioritizes cost over sustainability can adjust the weighting and immediately see how site rankings shift. This made the dashboard useful across different stakeholders with different priorities.
  • 05
    Dashboard Actions & Interactivity Configured Tableau dashboard actions so that selecting a region on one sheet filters all other sheets simultaneously. Clicking a bar in the regional comparison instantly updates the detail views — enabling a fluid top-down exploration flow without requiring the user to manually adjust multiple filters.

What the dashboard delivered

5+

Chart Types Deployed

Bar charts, dual-axis plots, tree maps, line charts, and a geographic map — each chosen to answer a specific analytical question for the site selection workflow.

Dynamic

Weighted Site Score

A parameter-driven composite score lets analysts adjust priority weights in real time — so cost-focused and sustainability-focused stakeholders can both use the same tool.

Multi

Sheet Architecture

Each dashboard sheet serves a distinct analytical purpose — geographic overview, regional comparison, site-level detail — connected by dashboard actions for seamless drill-down.

The final dashboard gave Intel's Sustainability Team a self-serve analytical tool — one that could answer "what if" questions about site selection criteria without requiring a new analysis for each scenario. The key design principle was that the tool should surface tradeoffs clearly rather than hiding complexity behind a single recommendation.

04 — Tableau Skills Demonstrated

Technical depth beyond basic charting

  • ✓
    Calculated fields and LOD expressions Going beyond simple aggregations to build derived metrics that answer specific business questions — composite scores, percentage-of-total calculations, and cross-dataset comparisons that can't be done in the source data alone.
  • ✓
    Parameter controls for user-driven analysis Parameters transform a static dashboard into a decision-support tool. Rather than presenting a single answer, the dashboard lets analysts explore the data according to their own priorities and assumptions.
  • ✓
    Dashboard actions for interactive storytelling Configuring filter, highlight, and URL actions turns a collection of charts into a coherent analytical workflow. Each interaction reveals more detail rather than requiring the user to navigate to a separate report.
  • ✓
    Chart type judgment The most important Tableau skill isn't knowing how to build any chart — it's knowing which chart to use for which question. Dual-axis for correlation, tree map for proportional comparison, line for trend — each choice was deliberate.
